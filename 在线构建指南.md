# 在线构建APK指南

## 已准备的文件
项目已准备好，位于 `lsp_for_upload` 文件夹中，包含：
- 完整的Android项目结构
- 源代码文件
- 构建配置文件
- 依赖配置
- 使用说明文档

## 推荐构建服务

### 1. AppCircle (推荐)
**网址**: https://appcircle.io/
**优点**: 免费额度充足，配置简单

**步骤**:
1. 注册/登录AppCircle账号
2. 点击"New App"
3. 选择"Upload Source Code"
4. 上传 `lsp_for_upload` 文件夹（或压缩为zip）
5. 选择Android构建
6. 使用默认配置
7. 点击"Start Build"

### 2. CodeMagic
**网址**: https://codemagic.io/
**优点**: 每月500分钟免费构建时间

**步骤**:
1. 注册/登录CodeMagic账号
2. 点击"Add application"
3. 选择"Build from source"
4. 上传项目文件
5. 选择Android项目类型
6. 构建命令: `./gradlew assembleDebug`
7. 点击"Start new build"

### 3. GitHub Actions (需要GitHub账号)
**优点**: 完全免费，集成度高

**步骤**:
1. 创建GitHub仓库
2. 上传 `lsp_for_upload` 文件夹内容
3. 创建 `.github/workflows/android.yml` 文件
4. 使用Android构建工作流
5. 触发构建

## 构建配置说明

### 依赖库
项目使用以下依赖，在线构建服务通常都支持：
- `de.robv.android.xposed:api:82` - Xposed框架API
- `org.json:json:20210307` - JSON处理库

### 构建命令
```bash
./gradlew assembleDebug
```

### 输出文件
构建成功后，下载 `app-debug.apk` 文件：
- 路径: `app/build/outputs/apk/debug/app-debug.apk`

## 注意事项

1. **Xposed模块**: 这是一个Xposed/LSPosed模块，需要在已root的设备上运行
2. **配置文件**: 模块会读取 `/data/adb/v2ray_autostart/config.json` 或 `/sdcard/v2ray_autostart_config.json`
3. **权限**: 需要root权限读取配置文件
4. **兼容性**: 支持Android 7.0+ (API 24+)

## 快速开始
1. 选择上述任一构建服务
2. 上传 `lsp_for_upload` 文件夹
3. 开始构建
4. 下载生成的APK文件
5. 安装到已root的Android设备
6. 在LSPosed中启用模块

## 故障排除
- **构建失败**: 检查构建日志，通常是依赖下载问题
- **APK无法安装**: 确保设备已root并安装LSPosed框架
- **模块不生效**: 检查配置文件路径和格式是否正确

构建完成后，你会得到一个可安装的APK文件，可以在支持Xposed/LSPosed的Android设备上使用。
